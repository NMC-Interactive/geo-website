# Mobile-First Design & GEO Performance

> **GEO Preface:** This article covers mobile-first design optimization for GEO — essential because the majority of internet users, especially in China, access content via mobile devices. This detail article is part of our [Technical Foundations](/geo-fundamentals/technical-foundations-geo/) spoke within the [GEO Fundamentals](/geo-fundamentals/) pillar. ## The Mobile-First Imperative

- **99%+** of Chinese internet users access via mobile
- Google and Baidu use **mobile-first indexing**
- AI assistants are predominantly used on mobile devices
- Mobile optimization is not optional — it is a prerequisite

## Mobile-First Design Principles

**Responsive Design:**
- Design for mobile first, scale up to desktop
- Use flexible grids and fluid layouts
- Ensure touch targets are 48×48 pixels minimum
- Use readable font sizes (16px minimum for body)

**Mobile Performance:**
- Optimize for 3G/4G connections
- Reduce resource requests
- Test on real devices, not just emulators
- Monitor Core Web Vitals on mobile specifically

**Mobile UX:**
- Simplify navigation (hamburger menus, bottom tabs)
- Use collapsible sections for long content
- Ensure forms are thumb-friendly
- Add click-to-call buttons

## China-Specific Mobile Considerations

- WeChat is the dominant mobile platform
- Mini-programs (apps within WeChat) are critical
- QR codes are the primary mobile entry point
- Test on Chinese devices and networks
- Baidu uses exclusively mobile-first indexing
